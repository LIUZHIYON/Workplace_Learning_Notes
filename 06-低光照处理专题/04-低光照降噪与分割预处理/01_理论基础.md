# 01 - 理论基础：低光照噪声与增强原理

## 1. 低光照图像的噪声模型

### 1.1 传感器噪声来源

低光照图像中的噪声主要由以下成分构成：

```
总噪声 ≈ 光子散粒噪声 + 读取噪声 + 暗电流噪声 + 量化噪声
```

| 噪声类型 | 来源 | 统计特性 | 与亮度的关系 |
|----------|------|----------|:----------:|
| **光子散粒噪声** (Shot Noise) | 光子到达的随机性 | 泊松分布 → 均方根 = √信号 | 信号越弱，信噪比越低 |
| **读取噪声** (Read Noise) | 电路读出 | 高斯分布 | 固定值，暗部显著 |
| **暗电流噪声** (Dark Current) | 传感器温度 | 泊松分布 | 温度相关 |
| **量化噪声** (Quantization) | ADC 采样 | 均匀分布 | 8bit 暗部仅 50 级 |

### 1.2 为什么低光照噪声严重

```
明亮场景 (信号~200):
    SNR = 200 / √200 ≈ 14.1  → 噪声不可见

低光场景 (信号~10):
    SNR = 10 / √10 ≈ 3.2     → 噪声明显！
```

**关键结论**: 低光照场景下 SNR 急剧下降，同样的传感器在暗部的有效信息极少。

## 2. 对比度拉伸为何放大噪声

### 2.1 数学解释

CLAHE / 直方图均衡化的本质是一个映射函数 $f(x)$:

$$y = f(x), \quad x \in [0, 255]$$

低光图像像素集中在 $[0, 50]$ 区间，$f(x)$ 将它拉伸到 $[0, 255]$。

假设原始噪声 $\sigma_x = 2$ (8bit)，映射后：

$$\sigma_y \approx |f'(x)| \cdot \sigma_x$$

当 $f(x)$ 把 $[0, 50] \to [0, 255]$ 时，$|f'(x)| \approx 5$，所以：

$$\sigma_y \approx 5 \times 2 = 10$$

即噪声被放大了 **5倍**！

### 2.2 可视化

```
原始低光直方图              拉伸后直方图
    |                          |
  |||  ████                   |  |  |  |  |  |  |
  |||  ████                   |  |  |  |  |  |  |
  |||  ████        ──→       |  |  |  |  |  |  |
  |||__████______              |__|__|__|__|__|__|
  0  50  100 255              0        128      255
                                  (稀疏 + 噪点多)
```

## 3. Retinex 理论简介

### 3.1 基本模型

Retinex 理论由 Edwin Land 提出，将图像分解为反射分量和照度分量的乘积：

$$I(x,y) = R(x,y) \cdot L(x,y)$$

- $I(x,y)$: 观察到的图像
- $R(x,y)$: 物体反射率 (内在属性，想恢复的)
- $L(x,y)$: 环境照度 (缓慢变化，可分离)

### 3.2 在低光照中的应用

$$\log R = \log I - \log L$$

通过估计照度 $L$ (大尺度高斯模糊)，可以恢复反射分量 $R$，从而增强暗部。

### 3.3 简化实践 — 单尺度 Retinex (SSR)

```python
def single_scale_retinex(img, sigma=80):
    img_log = np.log1p(img.astype(np.float32))
    L = cv2.GaussianBlur(img_log, (0, 0), sigma)  # 估计照度
    R = img_log - L  # 反射分量
    # 归一化到 [0, 255]
    R = cv2.normalize(R, None, 0, 255, cv2.NORM_MINMAX)
    return R.astype(np.uint8)
```

⚠️ Retinex 系列方法 (SSR/MSR/MSRCR) 虽然效果好，但 **计算量较大**，在 ARM 平台上更适合离线批处理。

## 4. 先降噪再增强的原理

### 4.1 降噪前置的核心逻辑

```
❌ 错误顺序: 增强 → 降噪
  噪声被放大 5× → 降噪器需要处理 5× 强度的噪声 → 效果打折

✅ 正确顺序: 降噪 → 增强
  噪声仍处于原始强度 → 降噪器效率最大化 → 增强干净信号
```

### 4.2 为什么在 L 通道处理

LAB 色彩空间的 L (Lightness) 通道独立于颜色信息：

- L 通道: 包含亮度和结构信息 → 在这里做增强/降噪
- A/B 通道: 包含色彩信息 → 不动它，色彩不失真

```python
lab = cv2.cvtColor(img, cv2.COLOR_BGR2LAB)
l, a, b = cv2.split(lab)
l_processed = denoise_and_enhance(l)
result = cv2.cvtColor(cv2.merge([l_processed, a, b]), cv2.COLOR_LAB2BGR)
```

## 5. 边缘保持降噪的数学直觉

### 5.1 高斯滤波的问题

高斯滤波对每个像素周围做等权平滑：

```
Edge: ▓▓▓▓░░░░░░    ← 明显的边界
        ↓ Gaussian blur
      ▓▓▒▒▒▒░░░░    ← 边界模糊了！
```

### 5.2 双边滤波的改进

双边滤波同时考虑 **空间距离** 和 **像素值差异**：

$$w(p,q) = \exp\left(-\frac{\|p-q\|^2}{2\sigma_s^2}\right) \cdot \exp\left(-\frac{\|I(p)-I(q)\|^2}{2\sigma_r^2}\right)$$

- $\sigma_s$: 空间权重 (距离近 → 权重高)
- $\sigma_r$: 范围权重 (像素值接近 → 权重高)

```
Edge: ▓▓▓▓░░░░░░
        ↓ Bilateral
      ▓▓▓▓░░░░░░    ← 边界保留了！
```

### 5.3 导向滤波的进一步改进

导向滤波由何恺明 (2010) 提出，核心假设是局部线性模型：

$$q_i = a_k I_i + b_k, \quad \forall i \in \omega_k$$

在窗口 $\omega_k$ 内，输出 $q$ 是引导图 $I$ 的线性变换。

**优势**: 时间复杂度 O(N)，与滤波核大小无关！
