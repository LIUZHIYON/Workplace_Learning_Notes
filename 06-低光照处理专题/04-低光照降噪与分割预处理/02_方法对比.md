# 02 - 方法详细对比

## 降噪方法

### 1. GaussianBlur — 高斯滤波

```python
cv2.GaussianBlur(img, (5, 5), sigmaX=1.0)
```

| 属性 | 值 |
|------|-----|
| 时间复杂度 | O(N·k²) (可分离卷积) |
| 720P CPU | ~1-2ms |
| 边缘保持 | ❌ 差 |
| 适用场景 | 仅做粗糙预处理 |

**评价**: 最基础的降噪，会把边缘一起糊掉。**不推荐用于分割前预处理**，除非你的分割模型本身对模糊鲁棒。

---

### 2. medianBlur — 中值滤波

```python
cv2.medianBlur(img, 3)  # 核大小 3
cv2.medianBlur(img, 5)  # 核大小 5
```

| 属性 | 值 |
|------|-----|
| 时间复杂度 | O(N·k²·log k) |
| 720P CPU | ~2-3ms (k=3), ~5-8ms (k=5) |
| 边缘保持 | ⭐⭐ 中等 |
| 椒盐噪声 | ✅ 极好 |

**评价**: 对椒盐噪声 (传感器坏点) 效果极佳，速度尚可。对高斯噪声效果一般。**适合作为快速预降噪的第一步**。

---

### 3. bilateralFilter — 双边滤波

```python
cv2.bilateralFilter(img, d=9, sigmaColor=75, sigmaSpace=75)
```

| 属性 | 值 |
|------|-----|
| 时间复杂度 | O(N·k²) (含 exp 运算) |
| 720P CPU | ~15-25ms (d=9), ~8-12ms (d=5) |
| 边缘保持 | ✅✅ 好 |
| 低光表现 | ⭐⭐ 中上 |

**参数调优 (低光专用)**:

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `d` | 5 | 核直径，低光用小核节省 CPU |
| `sigmaColor` | 50-75 | 颜色阈值，低光暗部差异小，不宜太大 |
| `sigmaSpace` | 50-75 | 空间阈值 |

**评价**: 边缘保持好，但非线性的指数运算在 ARM 上较重。适合中等算力场景。

---

### 4. fastNlMeansDenoising — 非局部均值降噪

```python
# 灰度
cv2.fastNlMeansDenoising(img, None, h=10, templateWindowSize=7, searchWindowSize=21)

# 彩色
cv2.fastNlMeansDenoisingColored(img, None, h=10, hColor=10, templateWindowSize=7, searchWindowSize=21)
```

| 属性 | 值 |
|------|-----|
| 时间复杂度 | O(N·S²·P²), S=搜索窗, P=模板窗 |
| 720P CPU | ~80-150ms (sw=21), ~40-80ms (sw=11) |
| 边缘保持 | ✅✅✅ 极好 |
| 低光表现 | ✅✅✅ 最佳 |
| ARM 适用性 | ⚠️ 需大幅缩小搜索窗 |

**参数调优 (ARM 低光专用)**:

| 参数 | 默认 | ARM 推荐 | 说明 |
|------|------|----------|------|
| `h` | 10 | 5-10 | 滤波强度 |
| `templateWindowSize` | 7 | 5 | 模板窗，必须是奇数 |
| `searchWindowSize` | 21 | 7-11 | **关键**: 缩小可大幅提速 |

**CPU 随 searchWindowSize 的变化 (720P 估算)**:

```
searchWindowSize=21:  ████████████████████  ~120ms
searchWindowSize=15:  █████████████▌        ~80ms
searchWindowSize=11:  ██████▌               ~50ms
searchWindowSize=7:   ███▌                  ~25ms
```

---

### 5. guidedFilter — 导向滤波 ⭐推荐

```python
cv2.ximgproc.guidedFilter(guide, src, radius=8, eps=0.01**2 * 255**2)
```

| 属性 | 值 |
|------|-----|
| 时间复杂度 | O(N) — 与核大小无关！ |
| 720P CPU | ~5-10ms (r=4~16 几乎相同) |
| 边缘保持 | ✅✅✅ 极好 (不输 NLM) |
| 低光表现 | ✅✅✅ 优秀 |
| ARM 适用性 | ✅✅✅ 首选 |

**参数调优**:

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `radius` | 4-16 | 滤波半径，越大平滑越强 |
| `eps` | 0.01²~0.04² | 正则化参数，越小保留越多边缘 |

**自引导模式**: 引导图 = 原图本身，无需额外输入。

```python
# 自引导降噪
denoised = cv2.ximgproc.guidedFilter(img, img, radius=8, eps=0.04**2 * 255**2)
```

**为何 O(N)**: 导向滤波基于盒式滤波 + 积分图，计算量完全独立于滤波核半径。这个特性让它在大核场景下碾压双边滤波。

## 对比度增强方法

### 6. CLAHE — 自适应直方图均衡

```python
clahe = cv2.createCLAHE(clipLimit=2.0, tileGridSize=(8,8))
result = clahe.apply(img)
```

| 属性 | 值 |
|------|-----|
| 720P CPU | ~2-4ms |
| 对比度提升 | ✅✅✅ 最强 |
| 噪声放大 | ❌ 会放大 (需配合降噪) |

**clipLimit 的影响**:

```
clipLimit=1.0:  温和增强，噪声放大少
clipLimit=2.0:  标准增强
clipLimit=4.0:  激进增强，噪声显著放大
```

**低光推荐**: `clipLimit=1.0~1.5`

---

### 7. Gamma 校正 (LUT 实现)

```python
gamma = 0.6
lut = np.array([((i/255.0)**gamma)*255 for i in range(256)], dtype=np.uint8)
result = cv2.LUT(img, lut)
```

| 属性 | 值 |
|------|-----|
| 720P CPU | ~0.5ms (LUT), ~3-5ms (逐像素 pow) |
| 暗部提升 | ✅✅ 好 |

**Gamma 值选择**:

```
gamma < 1:  提升暗部 (低光增强)
gamma = 1:  不变
gamma > 1:  压暗暗部
```

## 组合策略对比

| 策略 | 描述 | CPU (720P) | 降噪 | 增强 | 推荐 |
|------|------|:---:|:---:|:---:|:---:|
| `CLAHE only` | 仅增强不降噪 | 3ms | ❌ | ✅✅✅ | ❌ |
| `Gaussian → CLAHE` | 先糊再增 | 4ms | ⭐ | ✅✅✅ | ❌ |
| `Median → CLAHE` | 先除坏点再增 | 6ms | ⭐⭐ | ✅✅✅ | ⭐⭐ |
| `Bilateral → CLAHE` | 边保持降噪+增强 | 20ms | ⭐⭐ | ✅✅✅ | ⭐⭐⭐ |
| `Guided → CLAHE` | O(N) 降噪+增强 | 10ms | ⭐⭐⭐ | ✅✅✅ | ⭐⭐⭐⭐⭐ |
| `NLM → CLAHE` | 最强降噪+增强 | 60ms | ⭐⭐⭐ | ✅✅✅ | ⭐⭐⭐ |
| `Guided → Gamma → CLAHE` | 降噪+暗部提亮+局部增强 | 12ms | ⭐⭐⭐ | ✅✅✅ | ⭐⭐⭐⭐⭐ |
